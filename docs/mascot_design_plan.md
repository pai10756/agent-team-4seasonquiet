# 時時靜好 — 插畫吉祥物規劃

## 設計策略

採用 **Kurzgesagt 模式**：一隻固定插畫吉祥物作為每集「視覺錨點」，不需要動畫、不需要說話。

### 為什麼選插畫而非虛擬真人

| 虛擬真人（如陳阿姨） | 插畫吉祥物 |
|-------------------|-----------|
| Seedance 跨集會漂移 ~20% | 永遠 100% 一致 |
| 被識破 AI 時信任崩塌 | 本來就是畫的，無「造假」問題 |
| YouTube inauthentic policy 風險較高 | 明確是品牌視覺，不偽裝真人 |
| 只能用在影片裡 | 可延伸到 LINE 貼圖、縮圖、周邊 |

### 混合策略

插畫吉祥物負責**品牌辨識**，Seedance 虛擬真人（陳阿姨）繼續負責**正片內容**。

```
片頭（3秒）: 插畫吉祥物 + 頻道名 → 品牌辨識
     ↓
正片（30秒）: Seedance 虛擬真人 or 圖卡排行榜
     ↓
片尾（固定）: 插畫吉祥物 + 「時時靜好」logo → 記憶錨點
```

即使 Seedance 人物偶爾漂移，觀眾的品牌記憶鎖定在插畫角色上。

## 標竿案例

| 品牌 | 吉祥物 | 模式 | 啟發 |
|------|--------|------|------|
| **Kurzgesagt** | 黑鴨 Duck | 每集出現但不是主角，彩蛋式存在 | 視覺錨點型，與 33 秒 Shorts 最相容 |
| **Headspace** | The Dot / 150+ 角色系統 | 圓潤無尖角、柔和色彩傳達「安全放鬆」 | 視覺語言系統 > 單一角色 |
| **Duolingo** | Duo 貓頭鷹 | 角色有可辨識的「個性」 | 吉祥物最強狀態是有性格，不只可愛 |
| **台灣吧** | 黑啤 + 啤下組織 | 台灣黑熊 → LINE 貼圖 → 周邊 → 品牌合作 | 台灣市場驗證「本土動物 + 個性化 → IP 變現」 |
| **Woebot** | 圓臉機器人 | 用「萌」降低健康議題門檻 | 健康主題不需要嚴肅 |

## 角色概念方向

暫名：**「小靜」**（呼應「時時靜好」的「靜」）

### 方向 A：台灣藍鵲

| 元素 | 設計 |
|------|------|
| 物種 | 台灣藍鵲（台灣國鳥） |
| 配色 | 寶藍身體 + 紅嘴 + 白色腹部 |
| 配件 | 小圍裙或小菜籃 |
| 個性 | 好奇、微歪頭、愛分享知識 |
| 優勢 | 辨識度極高（長尾巴 + 藍色）、台灣國鳥自帶本土認同 |
| 風險 | 藍色調與頻道暖色溫可能需要平衡 |

### 方向 B：台灣樹蛙

| 元素 | 設計 |
|------|------|
| 物種 | 翡翠樹蛙 |
| 配色 | 翠綠身體 + 金色大眼 |
| 配件 | 頭上頂一片葉子（當帽子） |
| 個性 | 安靜、溫和、偶爾露出驚訝表情 |
| 優勢 | 綠色 = 自然健康感；圓潤身體天生萌 |
| 風險 | 部分觀眾可能不喜歡蛙類 |

### 方向 C：食材擬人 — 小南瓜

| 元素 | 設計 |
|------|------|
| 物種 | 擬人化南瓜 |
| 配色 | 暖橘色身體 + 綠色蒂頭（帽子） |
| 配件 | 圍裙、手拿小鍋鏟 |
| 個性 | 熱情、圓滾滾、笑瞇瞇 |
| 優勢 | 暖橘色完美匹配頻道色調；直接連結「飲食」主題 |
| 風險 | 南瓜有季節感（秋冬），需要確認全年適用性 |

### 方向 D：台灣石虎

| 元素 | 設計 |
|------|------|
| 物種 | 石虎（瀕危台灣特有亞種） |
| 配色 | 暖棕底 + 深色豹紋斑點 |
| 配件 | 小領巾或菜籃 |
| 個性 | 機靈、獨立、偶爾調皮 |
| 優勢 | 暖色系天然匹配；貓科自帶網路親和力；保育話題加分 |
| 風險 | 花紋細節多，極簡化時需要保留辨識度 |

## 設計規範

### 風格要求

- **圓潤扁平插畫**（Kurzgesagt 風格）
- 無尖角、無複雜陰影
- Gemini 可穩定生成的簡潔度
- 與頻道暖色調（橘、米、棕）協調

### 表情系統（至少 5 種）

1. 微笑（預設）
2. 驚訝（配合「認知反差」hook）
3. 思考（配合「你知道嗎」段落）
4. 開心（配合正面結論）
5. 揮手（片尾告別）

### 使用場景

| 場景 | 用法 |
|------|------|
| 影片片頭 | 吉祥物 + 頻道名動態出場（3 秒內） |
| 影片片尾 | 吉祥物揮手 + 「時時靜好」文字 |
| YouTube 縮圖 | 吉祥物在角落，搭配主題食材大圖 |
| 頻道 logo | 吉祥物正面微笑 |
| 未來延伸 | LINE 貼圖、IG 限動貼紙、周邊商品 |

### 資產清單（初期最小需求）

- [ ] 正面全身圖（標準 pose）
- [ ] 5 種表情頭像
- [ ] 透明背景 PNG（用於縮圖疊加）
- [ ] 片頭用橫幅（吉祥物 + 頻道名）
- [ ] 片尾用落款圖

## 製作方式

使用 Gemini 生成插畫風格角色，流程：

1. 用 Gemini prompt 生成多個方向的概念圖
2. 選定方向後，生成多角度定裝照（正面、側面、背面）
3. 生成 5 種表情變體
4. 匯出透明背景 PNG
5. 存入 `characters/mascot/` 資料夾，建立 `character.json`

### Gemini Prompt 範例（台灣藍鵲方向）

```
設計一隻可愛的台灣藍鵲卡通吉祥物，扁平插畫風格（flat illustration），
圓潤線條，無尖角。寶藍色身體，紅色嘴巴，白色腹部，長長的藍色尾巴。
穿著一件小小的米色圍裙，手（翅膀）拿著一個小菜籃。
表情是微微歪頭、好奇的眼神、溫暖的微笑。
背景純白，適合作為 YouTube 頻道吉祥物。
風格參考：Kurzgesagt 的圓潤扁平動物角色。
```

## 已建立的資產系統

| 檔案 | 用途 |
|------|------|
| `characters/mascot/character.json` | 角色規格 + 表情系統 + 使用規則（asset_gen 的 source of truth） |
| `characters/mascot/prompt_templates.md` | Gemini prompt 模板（縮圖、片頭、片尾、知識卡） |
| `characters/mascot/Duolingo_Real_leopard_cat.jpg` | 風格參考圖（style reference） |
| `schemas/episode.schema.json` → `mascot` 欄位 | scriptwriter 指定每集的表情和用途 |

### 生產流程

```
scriptwriter 在 episode JSON 中指定 mascot.thumbnail.expression = "surprised"
  → asset_gen 讀取 character.json 取得 surprised 的描述
  → asset_gen 用 prompt_templates.md 的縮圖模板組合完整 prompt
  → Gemini 生成透明背景 PNG
  → assembler 疊加到縮圖和影片中
```
